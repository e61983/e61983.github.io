在 Go 裡直接抓 Docker 容器 IP，連線 PostgreSQL

前言

開發環境裡，我們常用 Docker Compose 來啟動資料庫。像是 PostgreSQL 這種服務，我們通常會這樣做：

1ports:
2  - "5432:5432"

這樣就能用 localhost:5432 直接連資料庫，簡單又方便。

不過有些時候，我們不想把資料庫埠口對外暴露，想完全在 Docker network 裡面連線。
問題是，容器的 IP 會變啊，每次重啟都不一樣，怎麼辦？

答案是：別手動 docker inspect，直接在 Go 裡用 Docker API 查 IP 就好。
查完 IP，再用它去連 PostgreSQL，完全自動化，安全又乾淨。

Docker Compose 範例

最小化的 docker-compose.yaml 長這樣：

 1version: "3.9"
 2services:
 3  db:
 4    image: postgres:16
 5    restart: always
 6    environment:
 7      POSTGRES_USER: user
 8      POSTGRES_PASSWORD: password
 9      POSTGRES_DB: demo
10    volumes:
11      - db-data:/var/lib/postgresql/data
12
13volumes:
14  db-data:

注意：這裡沒有 ports:，資料庫只在 Docker network 裡對外提供服務。

Go 程式實作

我們會用兩個套件：

  • github.com/docker/docker/client → 查容器資訊
  • github.com/jackc/pgx/v5 → 連 PostgreSQL

程式流程很簡單：

  1. 先用 Docker API ContainerInspect 拿容器資訊
  2. 從 NetworkSettings.Networks 讀 IP
  3. 拼成連線字串，丟給 pgx 連資料庫

程式碼如下：

 1package main
 2
 3import (
 4	"context"
 5	"fmt"
 6	"log"
 7
 8	"github.com/docker/docker/client"
 9	"github.com/jackc/pgx/v5"
10)
11
12// 拿容器 IP
13func getContainerIP(ctx context.Context, containerName string) (string, error) {
14	cli, err := client.NewClientWithOpts(client.FromEnv)
15	if err != nil {
16		return "", fmt.Errorf("無法建立 Docker client: %w", err)
17	}
18	defer cli.Close()
19
20	containerJSON, err := cli.ContainerInspect(ctx, containerName)
21	if err != nil {
22		return "", fmt.Errorf("無法 inspect 容器: %w", err)
23	}
24
25	for _, network := range containerJSON.NetworkSettings.Networks {
26		return network.IPAddress, nil
27	}
28
29	return "", fmt.Errorf("找不到容器的 IP")
30}
31
32func main() {
33	ctx := context.Background()
34
35	ip, err := getContainerIP(ctx, "db")
36	if err != nil {
37		log.Fatalf("查 IP 失敗: %v", err)
38	}
39
40	fmt.Println("找到容器 IP:", ip)
41
42	connStr := fmt.Sprintf("postgres://user:password@%s:5432/demo", ip)
43
44	conn, err := pgx.Connect(ctx, connStr)
45	if err != nil {
46		log.Fatalf("無法連線資料庫: %v", err)
47	}
48	defer conn.Close(ctx)
49
50	var now string
51	err = conn.QueryRow(ctx, "SELECT now()").Scan(&now)
52	if err != nil {
53		log.Fatalf("查詢失敗: %v", err)
54	}
55
56	fmt.Println("成功連上資料庫，現在時間:", now)
57}

執行測試

  1. 先啟動 PostgreSQL 容器：
1docker compose up -d db
  1. 執行 Go 程式：
1go run main.go
  1. 會看到類似輸出：
1找到容器 IP: 172.19.0.2
2成功連上資料庫，現在時間: 2025-08-28 01:00:00.123456+00

注意事項

  • 這段程式會呼叫 Docker Daemon API，需要存取 /var/run/docker.sock。 如果 Go 程式也跑在容器裡，記得掛進去：
1volumes:
2  - /var/run/docker.sock:/var/run/docker.sock

  • 多個 network 時，程式會拿第一個 network 的 IP。實務上可以指定 network 名稱。

  • 容器 IP 只在 Docker network 裡有效，外部機器無法直接用。

小結

這樣做比直接開 ports: 麻煩一點，但更安全。
對內網環境或安全要求高的專案，這種方式非常適合。
